# Ghost v0.2.0 - Complete Desktop Automation Design
**Date:** 2026-04-17
**Status:** Approved
**Platform:** Windows-first
**Language:** Rust

---

## Overview

Ghost v0.2.0 extends the existing 3-crate workspace to deliver a complete 24-tool MCP vocabulary. Any AI agent (Claude, GPT-4, Gemini, open-source LLMs) can automate anything a human can do on a Windows computer. The agent provides intelligence and reasoning; Ghost provides execution primitives.

**v0.1.0 shipped:** 7 MCP tools (find, click, type, click_at, screenshot, launch, stop)
**v0.2.0 adds:** 17 MCP tools to reach full parity with human desktop capability

---

## Approach: Additive

Keep the 3-crate structure unchanged. Add new capabilities to existing crates. No new crates.

```
ghost-core      ← add new Win32 primitives
ghost-session   ← add thin session wrappers
ghost-mcp       ← add 17 new tool handlers
```

## Complete Tool Vocabulary (24 tools)

### Existing (7, unchanged)
| Tool | Description |
|------|-------------|
| `ghost_find` | Find element by name or role in UIA tree |
| `ghost_click` | Click a found element |
| `ghost_type` | Type text into a found element |
| `ghost_click_at` | Click at absolute screen coordinates |
| `ghost_screenshot` | Capture screen as PNG (base64) |
| `ghost_launch` | Launch a process by executable name |
| `ghost_stop` | Emergency stop, drain queue |

### New - Input (10)
| Tool | Parameters | Description |
|------|-----------|-------------|
| `ghost_press` | `key: string` | Press named key: Enter, Tab, Escape, Backspace, F1-F12, ArrowUp/Down/Left/Right, Home, End, PageUp, PageDown, Delete, Space |

| `ghost_key_up` | `key: string` | Release a held key |
| `ghost_hover` | `x: int`, `y: int` | Move mouse without clicking (triggers dropdowns, tooltips) |
| `ghost_right_click` | `x: int`, `y: int` | Right-click at coordinates |
| `ghost_double_click` | `x: int`, `y: int` | Double-click at coordinates |

| `ghost_scroll` | `x: int`, `y: int`, `direction: string`, `amount: int` | Scroll wheel (up/down/left/right) at coordinates |

### New - Perception (3)
| Tool | Parameters | Description |
|------|-----------|-------------|
| `ghost_describe_screen` | `window?: string` | Structured list of interactive elements (buttons, inputs, links, menus) with names, roles, bounding rects. Scoped to named window or full desktop. For non-vision LLMs. |
| `ghost_get_text` | `name: string`, `role?: string` | Get visible text or value from a found element (read field contents, labels, status bars) |
| `ghost_get_clipboard` | _(none)_ | Read current clipboard text |

### New - Window Management (3)
| Tool | Parameters | Description |
|------|-----------|-------------|

| `ghost_focus_window` | `name: string` | Bring window to foreground by partial name match |
| `ghost_window_state` | `name: string`, `state: string` | maximize / minimize / restore / close a window |

### New - Control Flow (1)
| Tool | Parameters | Description |
|------|-----------|-------------|

## Agent Perception-Action Loop

Any LLM can drive Ghost using this pattern for any arbitrary task:

```
1. ghost_describe_screen            → understand current state
2. ghost_find / ghost_click_at      → locate target
3. ghost_press / ghost_hotkey / ... → act
4. ghost_screenshot                 → verify (vision LLMs)
   OR ghost_get_text                → verify (text LLMs)
5. Repeat until task complete
```

Cross-app automation is first-class: the agent calls `ghost_list_windows`, `ghost_focus_window`, then operates - switching between apps exactly as a human would.

## Implementation Architecture

### ghost-core additions

**`input/keyboard.rs`**
- Add `key_down(vk: VIRTUAL_KEY)` - SendInput with 0 flags
- Add `key_up(vk: VIRTUAL_KEY)` - SendInput with KEYEVENTF_KEYUP
- Add `name_to_vk(name: &str) -> Option<VIRTUAL_KEY>` - map string names to VK codes

**`input/mouse.rs`**
- Add `hover(x: i32, y: i32)` - MOUSEEVENTF_MOVE only, no click
- Add `right_click(x: i32, y: i32)` - MOUSEEVENTF_RIGHTDOWN + MOUSEEVENTF_RIGHTUP
- Add `double_click(x: i32, y: i32)` - two click_events with ~50ms gap
- Add `drag(from_x, from_y, to_x, to_y)` - mousedown, move sequence, mouseup
- Add `scroll(x, y, direction, amount)` - MOUSEEVENTF_WHEEL (vertical) / MOUSEEVENTF_HWHEEL (horizontal)

**`uia/element.rs`**
- Add `get_text(element)` - IUIAutomationTextPattern2 → fallback to element name/value

**`uia/tree.rs`**
- Add `list_windows()` - EnumWindows, IsWindowVisible filter, GetWindowText
- Add `focus_window(name: &str)` - partial match on window titles, SetForegroundWindow
- Add `set_window_state(name: &str, state: WindowState)` - ShowWindow with SW_MAXIMIZE/MINIMIZE/RESTORE/CLOSE

**`system/clipboard.rs`** (new file)
- `get_clipboard() -> Result<String>` - OpenClipboard, GetClipboardData(CF_UNICODETEXT), CloseClipboard
- `set_clipboard(text: &str) -> Result<()>` - OpenClipboard, EmptyClipboard, SetClipboardData, CloseClipboard

**`system/mod.rs`** (new file) - expose clipboard module

### ghost-session additions

**`session.rs`**
- Thin wrappers for all new core primitives, same async pattern as existing methods
- `wait(ms: u64)` - tokio::time::sleep
- `wait_for(by: By, timeout_ms: u64)` - expose existing retry loop as public method
- `describe_screen(window: Option<&str>)` - query UIA tree, format element list as structured JSON

**`element.rs`**
- `get_text()` - sync call to core get_text (COM !Send constraint, same as existing click/type)

### ghost-mcp additions

**`main.rs`**
- 17 new tool handler match arms, same JSON-RPC pattern as existing 7
- `ghost_hotkey` parses `modifiers` array, maps each to VK, hold all then press key then release all
- `ghost_describe_screen` returns JSON array of element descriptors
- All tools return `{"ok": true}` or `{"error": "..."}` consistently

## Error Handling

No new error variants needed. All new tools map to existing errors:
- `Win32Error` - clipboard failures, ShowWindow failures, EnumWindows failures
- `ElementNotFound` - get_text on missing element
- `Timeout` - wait_for exceeds timeout
- `ghost_get_clipboard` returns empty string if clipboard is empty (not an error)
