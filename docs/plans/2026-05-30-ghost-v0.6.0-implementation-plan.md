### Task 0.1: Implement MCP `tools/call` (THE fix)

**Root cause:** `crates/ghost-mcp/src/main.rs` `handle()` routes raw tool names as JSON-RPC methods (`"ghost_click"`). The MCP spec requires `method: "tools/call"`, `params: { name, arguments }`, and the result wrapped in `content: [{type:"text", text}]`. `tools/list` works (tools appear registered) but the first real invocation returns `unknown method: tools/call` and a spec-compliant client (Claude Code/Desktop) closes the transport. `git log -S 'tools/call'` confirms this arm never existed in committed history (the 2026-04-17 memory fix was lost).

**Files:**
- Modify: `crates/ghost-mcp/src/main.rs` (`handle()` ~129-503, `McpResponse` ~29-37, error construction ~96-102)
- Test: same file `#[cfg(test)] mod tests`

**Step 1 — failing test:** add a test that a `tools/call` for an unknown tool returns a `content[]` with `isError:true` (NOT a transport error), and a known tool name routes through. Since `handle` needs a live session for most tools, test the *routing contract* via a pure helper `wrap_tool_result(Result<Value,(i64,String)>) -> Value`:
```rust
#[test]
fn tools_call_success_wraps_in_content_text() {
    let v = wrap_tool_result(Ok(json!({"ok":true})));
    assert_eq!(v["content"][0]["type"], "text");
    assert!(v["content"][0]["text"].as_str().unwrap().contains("ok"));
    assert!(v["isError"].is_null() || v["isError"]==json!(false));
}
#[test]
fn tools_call_error_wraps_as_iserror_content_not_transport_err() {
    let v = wrap_tool_result(Err((-32000,"boom".into())));
    assert_eq!(v["isError"], json!(true));
    assert!(v["content"][0]["text"].as_str().unwrap().contains("boom"));
}
```

**Step 3 — implement:**
- Add `fn wrap_tool_result(r: Result<Value,(i64,String)>) -> Value` producing `{content:[{type:"text",text:<json-or-msg>}], isError?:true}`.
- Split `handle()`: keep protocol arms (`initialize`, `initialized`, `tools/list`) in `handle()`; move all `ghost_*` arms into `async fn dispatch_tool(session, name, args) -> Result<Value,String>`.
- Add to `handle()`:
```rust
"tools/call" => {
    let name = p["name"].as_str().ok_or("tools/call missing 'name'")?.to_string();
    let args = p.get("arguments").cloned().unwrap_or(json!({}));
    let out = dispatch_tool(session, &name, &args).await;
    Ok(wrap_tool_result(out.map_err(|e| (-32000, e))))   // never Err → transport stays up
}
```
- Keep the legacy raw-name arm as a fallback (`other => dispatch_tool(session, other, &p).await.map(|v| v)`) so comet-mcp and existing tests keep working.

**Step 5 — commit:** `fix(mcp): implement tools/call with content[] wrapping (transport-killer)`

### Task 0.4: Focus-safe `focus_window` with verify+retry

**Root cause:** `crates/ghost-core/src/uia/tree.rs:279-289` `focus_window` is fire-and-forget `SetForegroundWindow` — no confirmation, no retry, no foreground-steal workaround. Keyboard ops that follow land in the wrong window.

**Files:** Modify `crates/ghost-core/src/uia/tree.rs` (`focus_window`); add helper `ensure_foreground(hwnd, timeout_ms) -> Result<bool>`.

**Step 3 — implement** the standard Windows foreground-steal workaround:
```rust
pub fn ensure_foreground(hwnd: HWND, timeout_ms: u64) -> Result<bool, CoreError> {
    unsafe {
        if GetForegroundWindow() == hwnd { return Ok(true); }
        // Attach to the foreground thread's input queue to gain SetForegroundWindow rights.
        let fg = GetForegroundWindow();
        let cur_tid = GetCurrentThreadId();
        let fg_tid = GetWindowThreadProcessId(fg, None);
        let tgt_tid = GetWindowThreadProcessId(hwnd, None);
        let _ = AttachThreadInput(cur_tid, fg_tid, TRUE);
        let _ = AttachThreadInput(cur_tid, tgt_tid, TRUE);
        let _ = ShowWindow(hwnd, SW_SHOW);
        let _ = BringWindowToTop(hwnd);
        let _ = SetForegroundWindow(hwnd);
        let _ = AttachThreadInput(cur_tid, fg_tid, FALSE);
        let _ = AttachThreadInput(cur_tid, tgt_tid, FALSE);
        let start = std::time::Instant::now();
        while start.elapsed().as_millis() < timeout_ms as u128 {
            if GetForegroundWindow() == hwnd { return Ok(true); }
            std::thread::sleep(std::time::Duration::from_millis(15));
        }
        Ok(GetForegroundWindow() == hwnd)
    }
}
```
`focus_window` calls `ensure_foreground(hwnd, 600)`, returns `CoreError` (new variant `FocusFailed{window}`) if it can't confirm. Tests: unit test name-resolution failure path; live integration test (Notepad) asserts foreground after focus. **Commit:** `fix(core): focus_window verifies foreground with AttachThreadInput retry`
