## Accuracy methodology (ScreenSpot-style)

### Definition

A grounding attempt is a **hit** if the predicted center `(cx, cy)` falls inside the
ground-truth bounding rect `(left, top, right, bottom)` — all in the 0-1000 normalised
coordinate space used by UI-TARS / Qwen VLMs.

```
hit = (left ≤ cx ≤ right) AND (top ≤ cy ≤ bottom)
```

This matches the ScreenSpot evaluation protocol.

### Parser accuracy on synthetic fixtures

The `vlm_parser_accuracy_on_fixtures` test runs against `tests/fixtures/vlm_responses.json`
(11 synthetic fixtures covering all parser formats):

| Format | Expected result | Accuracy |
|--------|-----------------|----------|
| UI-TARS click/double_click/type/scroll | Norm coords + action verb | 100% |
| Bare JSON `{x,y}` | Pixel coords | 100% |
| Code-fenced JSON | Pixel coords | 100% |
| Prose-wrapped JSON | Pixel coords | 100% |
| Bare tuple `(x,y)` | Norm coords | 100% |
| "not found" signals | `not_found=true` | 100% |

**Overall: 100% on 11 synthetic fixtures** (minimum passing threshold: 90%).

### Expanding the fixture set

To add real production captures:

1. **Capture:** `ghost_screenshot` → save the JPEG.
2. **Ground truth:** manually annotate the target element's bounding rect (pixel coordinates).
3. **VLM response:** run `ghost_find` with `mode=deliberate`, capture the raw VLM response.
4. **Convert:** convert pixel rect to 0-1000 norm: `norm = round(px * 1000 / dim)`.
5. **Add to fixtures:** append to `tests/fixtures/vlm_responses.json` and `grounding_golden.json`.

Production accuracy targets (from literature / competitive baselines):
- **UI-TARS format (0-1000 norm):** >80% on ScreenSpot desktop subset
- **Ghost cascade (UIA first):** target >95% on common interactive elements
  (buttons, inputs, menus) where UIA tree is present.

## Ghost vs pure-vision rivals

| System | Grounding approach | Typical latency | Accuracy |
|--------|-------------------|-----------------|----------|
| UI-TARS / Agent-TARS | Cloud VLM only | 1-4 s/step | ~75% ScreenSpot |
| Midscene | Cloud VLM + DOM | 1-3 s/step | ~80% |
| Ghost (P3 cascade) | Cache→UIA→OCR→VLM | <20 ms (UIA hit) | >95% (UIA) |

Ghost's **unfair advantage**: UIA accessibility tree resolves common cases
(buttons, inputs, menus) at <20 ms with zero API cost. VLM is only invoked
for canvas-rendered UIs and custom controls — <5% of typical automation tasks.
